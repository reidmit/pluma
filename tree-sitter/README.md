# tree-sitter-pluma

A [Tree-sitter](https://tree-sitter.github.io) grammar for the
[Pluma](https://pluma.fun) language (`.pa` files).

Tree-sitter is the highlighting (and structural-editing) engine for Helix,
Neovim, Zed, Emacs, and GitHub code navigation. Unlike the TextMate/Sublime
grammars (which only color text), this produces a real parse tree.

## How it handles Pluma's quirks

- **kebab-case identifiers** (`to-string`, `this-is-it`) are one token, and
  keywords only fire as whole words — both fall out of `word: $.identifier` and
  the lexer's longest-match rule. No lookarounds needed.
- **Statement separation.** Pluma has no `;`; statements are newline-separated.
  `src/scanner.c` is a small external scanner that emits a `_newline` token only
  where the parser expects a terminator (`valid_symbols`). Inside `()`/`[]` and
  right after a binary operator a newline isn't a valid terminator, so it's
  skipped — line continuation works for free, with no indentation tracking
  (Pluma blocks are brace-delimited).
- Precedence mirrors `compiler/src/operator.rs`.

## Layout

| File | What |
| --- | --- |
| `grammar.js` | the grammar |
| `src/scanner.c` | external scanner (the `_newline` terminator) |
| `src/parser.c` | generated by `tree-sitter generate` (committed for consumers) |
| `queries/highlights.scm` | highlight queries |
| `script/test.sh` | parses every `tests/run/*` fixture; fails on any ERROR |

## Develop

```sh
npm install                 # gets tree-sitter-cli
npx tree-sitter generate    # regenerate src/parser.c after editing grammar.js
npx tree-sitter parse ../tests/run/hello/main.pa   # inspect a parse tree
./script/test.sh            # regression: parse the whole tests/run corpus
```

From the repo root, `just test-grammar` runs this alongside the TextMate tests.

## Editor setup

- **Neovim** (nvim-treesitter): register `pluma` pointing at this directory, map
  `.pa` to it, and install the `highlights.scm` query.
- **Helix**: add a `[[language]]` for `pluma` and a `[[grammar]]` with this repo
  as the source in `languages.toml`, then `hx --grammar build`.
