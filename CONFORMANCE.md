# Pluma runtime conformance

Pluma compiles one IR to two backends. The **VM** is the reference/oracle; the
**WasmGC** deploy backend (server/compute, and the browser/client target as the
frontend lands) is run on every execution fixture — `tests/run` (happy path) +
`tests/run-fail` (compiles, fails at runtime) — and diffed against the VM. This file
is the living coverage matrix, regenerated by `just conformance` (perf numbers live
in `target/conformance/`, not here).

Corpus: **219** execution fixtures across `tests/run` + `tests/run-fail`. Compile-
error cases are excluded — they belong to the frontend `tests/analyze` suite.
